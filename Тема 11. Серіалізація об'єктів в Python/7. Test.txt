1. Що таке серіалізація об'єктів в Python?

!- Перетворення об'єктів Python у байтові рядки
-- Копіювання об'єктів у пам'яті
-- Збільшення швидкості виконання коду
-- Зменшення розміру файлу

Вірно! Серіалізація перетворює об'єкти Python у такий формат, що може бути збережений у файл або переданий по мережі.

2. Яка основна мета серіалізації?

-- Оптимізація виконання програми
-- Шифрування даних програми
!- Зберегти стан об'єкта
-- Архівування даних програми

Вірно! Основна мета серіалізації - зберегти стан об'єкта для подальшого відновлення або передачі.

3. Чим відрізняється серіалізація за допомогою модуля pickle від JSON?

-- серіалізація JSON швидша за pickle
!- модуль pickle може серіалізувати майже всі об'єкти Python
-- JSON може серіалізувати більше типів даних
-- модуль pickle є безпечнішим за JSON

Формат JSON є більш універсальним для веб-застосунків, але pickle підтримує більш широкий спектр типів Python.

4. Що точно ми не можемо серіалізувати за допомогою JSON?

!- Власні класи
-- Числові дані
-- Списки та словники
-- Можемо серіалізувати всі типи даних

JSON обмежений у серіалізації власних класів Python без їх перетворення.

5. Яким методом можна десеріалізувати об'єкт з файлу згідно модуля pickle?

-- Використовуючи метод write()
-- Використовуючи метод read()
!- Використовуючи метод load()
-- Використовуючи метод save()

Вірно! Метод load() використовується для десеріалізації об'єктів із файлу.
